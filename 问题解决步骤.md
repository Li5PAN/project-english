# 路径别名问题解决步骤

## 问题原因
Vite 无法解析 `@` 别名，因为 `vite.config.js` 中没有配置路径别名。

## 已完成的修复
✅ 已在 `vite.config.js` 中添加了路径别名配置：
```javascript
resolve: {
  alias: {
    '@': path.resolve(__dirname, './src')
  }
}
```

## 需要执行的步骤

### 1. 停止当前运行的开发服务器
在终端中按 `Ctrl + C` 停止服务器

### 2. 重新启动开发服务器
```bash
pnpm run dev
```

### 3. 清除浏览器缓存（可选）
如果还有问题，可以：
- 按 `Ctrl + Shift + R` 强制刷新页面
- 或者在开发者工具中勾选 "Disable cache"

## 验证是否成功

启动后应该能看到：
```
VITE v7.x.x  ready in xxx ms

➜  Local:   http://localhost:3000/
➜  Network: use --host to expose
```

访问 `http://localhost:3000/#/login` 应该能正常显示登录页面，不再报错。

## 如果还有问题

### 方案1：检查 Node.js 版本
确保 Node.js 版本 >= 16

```bash
node -v
```

### 方案2：清除缓存并重新安装依赖
```bash
# 删除 node_modules 和 lock 文件
rm -rf node_modules pnpm-lock.yaml

# 重新安装
pnpm install

# 启动
pnpm run dev
```

### 方案3：使用相对路径（临时方案）
如果路径别名还是不工作，可以临时修改 `login.vue` 中的导入：

```javascript
// 将
import { login, getUserInfo } from '@/api/auth'
import { message } from 'ant-design-vue'

// 改为
import { login, getUserInfo } from '../api/auth'
import { message } from 'ant-design-vue'
```

但这不是推荐的方案，应该优先解决路径别名配置问题。
