# 登录调试指南

## 当前问题分析

根据错误信息：
```
登录失败: Error: 请求失败
    at request.js:42:29
```

这表示后端返回的响应不符合预期的格式，被响应拦截器拦截了。

## 调试步骤

### 1. 打开浏览器开发者工具
按 `F12` 打开开发者工具

### 2. 切换到 Network 标签页
- 确保 Network 标签页是打开的
- 勾选 "Preserve log"（保留日志）

### 3. 尝试登录
输入账号密码，点击登录按钮

### 4. 查看 /login 请求详情

在 Network 列表中找到 `/login` 请求，点击查看详情：

#### 4.1 查看 Request（请求）
- **Request URL**: 应该是 `http://106.52.154.12:58080/login`
- **Request Method**: 应该是 `POST`
- **Request Payload**: 应该包含
  ```json
  {
    "username": "你输入的账号",
    "password": "你输入的密码"
  }
  ```

#### 4.2 查看 Response（响应）
这是最关键的部分，查看后端实际返回的数据格式。

**可能的情况：**

**情况1：后端返回成功但格式不同**
```json
{
  "code": 1,  // 不是 200
  "msg": "登录成功",
  "token": "xxx",
  "data": {...}
}
```

**情况2：后端返回错误**
```json
{
  "code": 500,
  "msg": "账号或密码错误"
}
```

**情况3：后端返回 HTML（说明接口地址错误）**
```html
<!DOCTYPE html>
<html>...
```

**情况4：网络错误（CORS 或连接失败）**
- Status: (failed) 或 CORS error
- 没有 Response 数据

### 5. 查看 Console 日志

现在代码中添加了详细的日志，在 Console 中可以看到：

```javascript
API 响应: {...}  // 后端返回的完整数据
登录响应完整数据: {...}  // 登录接口返回的数据
解析的用户信息: {...}  // 解析出的用户信息
用户角色: "student"  // 识别的角色
```

如果登录失败，会看到：
```javascript
业务错误: "xxx" {...}  // 业务层面的错误
登录失败详细信息: {...}  // 完整的错误信息
```

## 常见问题及解决方案

### 问题1：后端返回的 code 不是 200

**现象：** Console 显示 `业务错误: 请求失败`

**原因：** 后端使用的成功状态码不是 200

**解决方案：** 查看后端实际返回的 code 值，然后修改 `src/utils/request.js`

例如，如果后端成功时返回 `code: 1`，修改为：
```javascript
if (res.code === 1 || res.code === '1') {
  return res
}
```

### 问题2：CORS 跨域错误

**现象：** Network 显示 CORS policy 错误

**解决方案1：** 让后端配置 CORS

**解决方案2：** 使用代理（已配置）
修改 `src/utils/request.js` 的 baseURL：
```javascript
baseURL: '/api'  // 使用代理
```

### 问题3：网络连接失败

**现象：** Network 显示 (failed)

**原因：** 
- 后端服务未启动
- 后端地址错误
- 网络不通

**解决方案：**
1. 确认后端服务是否运行
2. 在浏览器直接访问 `http://106.52.154.12:58080` 测试连通性
3. 检查防火墙设置

### 问题4：接口地址错误

**现象：** 返回 404 或 HTML 页面

**解决方案：** 确认后端登录接口的正确地址

可能的接口地址：
- `/login`
- `/api/login`
- `/auth/login`
- `/user/login`

### 问题5：请求参数格式错误

**现象：** 后端返回参数错误

**解决方案：** 确认后端需要的参数格式

可能需要的格式：
```javascript
// 格式1：直接传递
{ username: "xxx", password: "xxx" }

// 格式2：嵌套在 user 对象中
{ user: { username: "xxx", password: "xxx" } }

// 格式3：使用不同的字段名
{ account: "xxx", pwd: "xxx" }
```

if (res.code === 1 || res.code === '1') {

```javascript
const userInfo = res.userInfo || res.data || res.user || res

```

## 快速测试后端接口

在浏览器 Console 中执行：

```javascript
fetch('http://106.52.154.12:58080/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    username: 'student',
    password: '123456'
  })
})
.then(res => res.json())
.then(data => {
  console.log('后端返回格式:', data)
  console.log('code 类型:', typeof data.code, '值:', data.code)
  console.log('是否有 token:', !!data.token)
  console.log('用户信息位置:', data.data || data.user || data.userInfo)
})
.catch(err => {
  console.error('请求失败:', err)
})
```

这会直接测试后端接口，并显示返回的数据格式。
