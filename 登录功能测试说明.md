# 登录功能测试说明

## 一、前置条件检查

### 1. 确认后端服务运行
后端地址：`http://106.52.154.12:58080`

可以通过以下方式测试后端是否可访问：
```bash
# 在浏览器控制台或使用 curl 测试
curl http://106.52.154.12:58080/login
```

### 2. 确认依赖已安装
```bash
cd English-study
pnpm install
```

### 3. 启动前端项目
```bash
pnpm run dev
```

项目会在 `http://localhost:3000` 启动

## 二、登录测试步骤

### 1. 访问登录页面
打开浏览器访问：`http://localhost:3000/#/login`

### 2. 输入测试账号
根据后端提供的测试账号输入，例如：
- 账号：`student`
- 密码：`123456`（或后端提供的密码）

### 3. 点击登录按钮

### 4. 观察结果

**成功情况：**
- 显示"登录成功"提示
- 自动跳转到学生端首页 `/student`
- 浏览器 localStorage 中保存了 token 和 userInfo

**失败情况：**
- 显示错误提示信息
- 停留在登录页面

## 三、调试方法

### 1. 打开浏览器开发者工具
按 `F12` 或右键 → 检查

### 2. 查看 Network 请求
- 切换到 Network 标签页
- 点击登录按钮
- 查看 `/login` 请求的详细信息

**检查项：**
- Request URL: 应该是 `http://106.52.154.12:58080/login`
- Request Method: 应该是 `POST`
- Request Payload: 应该包含 username 和 password
- Response: 查看后端返回的数据格式

### 3. 查看 Console 日志
- 切换到 Console 标签页
- 查看是否有错误信息
- 登录失败时会打印 `登录失败:` 相关信息

### 4. 查看 localStorage
- 切换到 Application 标签页
- 左侧选择 Storage → Local Storage
- 查看是否保存了 `token` 和 `userInfo`

## 四、常见问题排查

### 问题1：跨域错误 (CORS)
**现象：** Console 显示 CORS policy 错误

**解决方案：**
需要后端配置允许跨域，或者在前端配置代理。

在 `vite.config.js` 中添加代理配置：
```javascript
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://106.52.154.12:58080',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')
      }
    }
  }
})
```

然后修改 `src/utils/request.js` 中的 baseURL：
```javascript
baseURL: '/api'
```

### 问题2：网络连接失败
**现象：** 提示"网络连接失败，请检查网络"

**排查：**
1. 检查后端服务是否启动
2. 检查网络连接
3. 尝试直接访问后端地址

### 问题3：401 未授权
**现象：** 提示"未授权，请重新登录"

**原因：** 账号密码错误或后端验证失败

**解决：** 确认使用正确的测试账号

### 问题4：响应数据格式不匹配
**现象：** 登录成功但无法跳转，或提示角色未识别

**排查：**
1. 在 Console 中查看后端返回的数据格式
2. 检查 `login.vue` 中的数据解析逻辑是否匹配

**调试代码：**
在 `login.vue` 的 `handleLogin` 函数中添加：
```javascript
console.log('登录响应:', res)
console.log('用户信息:', userInfo)
console.log('角色:', role)
```

### 问题5：角色跳转错误
**现象：** 登录成功但跳转到错误的页面

**原因：** 后端返回的角色字段与前端判断不匹配

**解决：** 根据后端实际返回的角色值调整 `login.vue` 中的判断逻辑

## 六、验证登录成功的标志

### 1. 页面跳转
- 学生账号 → `/student` 页面
- 教师账号 → `/teacher` 页面
- 管理员账号 → `/admin` 页面

### 2. localStorage 存储
打开开发者工具 → Application → Local Storage，应该看到：
- `token`: JWT token 字符串
- `userInfo`: 用户信息 JSON 字符串

### 3. 页面显示
- 顶部导航栏显示用户名
- 侧边栏显示对应角色的菜单

## 七、快速测试命令

在浏览器 Console 中执行以下代码快速测试登录：

```javascript
// 测试登录接口
fetch('http://106.52.154.12:58080/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    username: 'student',
    password: '123456'
  })
})
.then(res => res.json())
.then(data => {
  console.log('登录响应:', data)
})
.catch(err => {
  console.error('登录失败:', err)
})
```
